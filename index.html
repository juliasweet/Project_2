Extract, Transform, Load

The purpose of this project was to gather data and create added value to the data, through the process of extracting, transforming, and loading. We sought to take the data around major weather diasters and combine it in to a more user friendly source of data. FEMA proved to be a good source to gather the data, however the layout of the data was cumbersome. Not only did it contain information that would only be relevant to FEMA, but there was no good way to use parts from each source without doing it manually. Our goal was to change that, and to accomplish that, we took the following steps.


Summary of work completed for the ETL Project:
 •Identified data sources
 •Selected 3 data sources from the FEMA.gov: 2 CSVs files and .json file:
    • FEMA Web Declaration Areas
    • FEMA Web Disaster Declarations
    • FEMA Web Disaster Summaries
 •Used Pandas to load 2 CSVs and 1 Json file, read files, eliminated unnecessary
 columns and made copy of each file, dropped "hour/minutes/second/timezones",
 renamed column names to lower case column names
 •Used Pandas to export transformed files into PostgreSQL and created tables.
 •Created SQL queries in PostgreSQL to show the benefit of having all data in one
 established database.
